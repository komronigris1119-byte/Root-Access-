## KIRISH

Hozirgi kunda axborot texnologiyalarining rivojlanishi har bir soha, jumladan qurilish sanoatiga ham katta ta'sir ko'rsatmoqda. Qurilish kompaniyalari o'z xizmatlarini zamonaviy raqamli platformalar orqali taqdim etish zaruriyati kuchayib bormoqda. Internet orqali mijozlar bilan muloqot qilish, loyihalarni namoyish etish va xizmatlarni reklama qilish bugungi kunda biznesning muhim qismi hisoblanadi.

### Mavzuning dolzarbligi

Qurilish sohasida raqobatning kuchayishi kompaniyalarni yangi mijozlarni jalb qilish va mavjud mijozlar bilan munosabatlarni mustahkamlash uchun innovatsion yondashuvlarni qo'llashga majbur qilmoqda. Web sayt - bu kompaniyaning raqamli yuzasi bo'lib, u quyidagi muhim vazifalarni bajaradi:

1. **Brend identifikatsiyasi va obro'** - Professional web sayt kompaniyaning ishonchliligini oshiradi va zamonaviy mijozlar orasida ijobiy taassurot qoldiradi.

2. **Marketing va reklama samaradorligi** - Onlayn mavjudlik an'anaviy reklama usullariga qaraganda ancha tejamkor va keng auditoriyaga yetib borish imkonini beradi.

3. **Mijozlar bilan bevosita aloqa** - Web sayt mijozlarga 24/7 kompaniya xizmatlari haqida ma'lumot olish, so'rov yuborish va maslahat olish imkoniyatini beradi.

4. **Loyihalar portfeli namoyishi** - Qurilish kompaniyasi o'zining bajarilgan ishlarini vizual tarzda namoyish etish orqali o'z malakasini namoyon qilishi mumkin.

5. **Raqobatbardoshlik** - Zamonaviy mijozlar, ayniqsa yosh avlod, birinchi navbatda internetda ma'lumot qidiradi. Web saytga ega bo'lmagan kompaniyalar bu mijozlarni yo'qotib qo'yishi mumkin.

O'zbekiston Respublikasida qurilish sohasi jadal sur'atlar bilan rivojlanmoqda. Prezidentimizning "Raqamli O'zbekiston - 2030" strategiyasiga muvofiq, barcha sohalar raqamlashtirilmoqda. Qurilish kompaniyalari ham bu jarayondan chetda qolmasligi lozim.

### Loyihaning maqsadi

Ushbu loyihaning asosiy maqsadi - zamonaviy texnologiyalar va dizayn tamoyillariga asoslangan, foydalanuvchilarga qulay va funksional qurilish kompaniyasi web saytini ishlab chiqishdir.

### Loyihaning vazifalari

Maqsadga erishish uchun quyidagi vazifalar belgilangan:

1. **Nazariy tadqiqotlar:**
   - Zamonaviy web texnologiyalarni o'rganish
   - Qurilish sohasidagi web saytlar tajribasini tahlil qilish
   - UX/UI dizayn tamoyillarini chuqur o'rganish
   - SEO va digital marketing strategiyalarini tahlil qilish

2. **Tahliliy ishlar:**
   - Maqsadli auditoriyani aniqlash va tahlil qilish
   - Funksional talablarni yig'ish va hujjatlashtirish
   - Raqobatchilar saytlarini tahlil qilish
   - Optimal texnologiyalar to'plamini tanlash

3. **Loyihalash:**
   - Ma'lumotlar bazasi strukturasini ishlab chiqish
   - Sayt arxitekturasini loyihalash
   - Interfeys dizayni va foydalanuvchi tajribasini loyihalash
   - Xavfsizlik mexanizmlarini rejalashtirish

4. **Amaliy ishlar:**
   - Frontend qismini dasturlash (HTML, CSS, JavaScript)
   - Backend tizimini yaratish
   - Ma'lumotlar bazasini joriy etish
   - CMS (Content Management System) ni integratsiya qilish
   - Adaptiv dizaynni amalga oshirish

5. **Sinovlar va optimallashtirish:**
   - Funksionallik testlari o'tkazish
   - Turli qurilmalarda ishlashini tekshirish
   - Yuklanish tezligini optimallashtirish
   - SEO optimizatsiyasi



### Tadqiqot ob'ekti va predmeti

**Tadqiqot ob'ekti:** Qurilish kompaniyasi uchun web sayt ishlab chiqish jarayoni va zamonaviy veb-texnologiyalar.

**Tadqiqot predmeti:** Web sayt yaratishning zamonaviy metodologiyalari, dasturlash tillari, freymvorklar va qurilish sohasiga xos funksional talablar.

### Tadqiqot metodlari

Loyihani amalga oshirishda quyidagi metodlardan foydalanildi:

1. **Nazariy tahlil metodi** - mavjud adabiyotlar, internet manbalar va ilmiy ishlarni o'rganish
2. **Qiyosiy tahlil** - o'xshash loyihalardagi eng yaxshi tajribalarni o'rganish
3. **Tizimli yondashuv** - web saytni yaxlit tizim sifatida loyihalash
4. **Eksperimental metod** - turli texnologiyalarni sinab ko'rish va taqqoslash
5. **Modellashtirish** - ma'lumotlar bazasi va arxitektura modellarini yaratish

### Loyihaning ilmiy-amaliy ahamiyati

**Amaliy ahamiyati:**
- To'liq funksional web saytning tayyor yechimini taqdim etish

- Mijozlar bilan muloqotni yaxshilash va yangi buyurtmalar sonini oshirish

### Kutilayotgan natijalar

Loyiha yakunida quyidagi natijalar kutilmoqda:

1. **Texnik jihatdan:**
   - Zamonaviy texnologiyalarda ishlab chiqilgan to'liq funksional web sayt
   - Adaptiv (responsive) dizayn - barcha qurilmalarda to'g'ri ishlash
   - Yuqori ishlash tezligi va optimallashtirilgan kod
   - Xavfsiz va ishonchli tizim

2. **Funksional jihatdan:**
   - Loyihalar galereyasi va portfolio
   - Xizmatlar katalogi
   - Onlayn so'rov va shartnoma tuzish tizimi
   - Kompaniya yangiliklari va blog
   - Mijozlar uchun shaxsiy kabinet
   - Administrator paneli (CMS)

3. **Biznes jihatdan:**
   - Kompaniya brendini mustahkamlash
   - Mijozlar sonini oshirish
   - Marketing xarajatlarini kamaytirish
   - Biznes jarayonlarini avtomatlashtirish

**Xulosa qismida** olingan natijalar umumlashtirilgan va takliflar berilgan.

## 1-BOB. LOYIHA HAQIDA UMUMIY MA'LUMOTLAR

### 1.1. Mavzuning dolzarbligi va ahamiyati

XXI asr - axborot texnologiyalari asri hisoblanadi. Bugungi kunda har qanday biznes, xizmat yoki mahsulot internetda o'z ifodasini topishi kerak. Qurilish sohasi ham bundan mustasno emas. Global raqamlashtirish jarayoni qurilish kompaniyalarini ham o'z ichiga qamrab olmoqda.

#### 1.1.1. Global tendentsiyalar

Jahon amaliyotida qurilish kompaniyalarining 95% dan ortig'i o'z web saytiga ega. Bu tasodifiy emas:

**Statistik ma'lumotlar shuni ko'rsatadiki:**
- Yangi mijozlarning 87% birinchi navbatda internetda qidiruv qiladi
- Qurilish xizmatlariga ehtiyoji bo'lgan kishilarning 72% uchta yoki undan ortiq saytlarni solishtirib ko'radi
- Professional web saytga ega bo'lgan kompaniyalar 40% ko'proq murojaat oladi
- Onlayn marketing an'anaviy usullarga qaraganda 3 marta tejamkorroq

**Raqamli transformatsiyaning afzalliklari:**

1. **Doimiy mavjudlik** - Web sayt 24/7 ishlaydi, dam olish kunlari va bayramlarda ham mijozlar ma'lumot olishlari mumkin.

2. **Geografik cheklovlarning yo'qligi** - Internet orqali butun mamlakat, hatto xalqaro mijozlarga yetib borish mumkin.

3. **Arzon marketing** - An'anaviy reklama (televideniye, gazeta, bilbordlar)ga qaraganda web marketing ancha arzon va samaraliroq.

4. **Mijozlar bazasini shakllantirish** - Sayt orqali mijozlar ma'lumotlarini to'plash va keyinchalik ular bilan ishlash mumkin.

5. **Brendni rivojlantirish** - Professional web sayt kompaniya imidjini sezilarli darajada oshiradi.

#### 1.1.2. O'zbekistonda raqamlashtirish

O'zbekiston Respublikasi Prezidentining "Raqamli O'zbekiston - 2030" strategiyasi doirasida barcha sohalarni raqamlashtirish davlat siyosatining ustuvor yo'nalishi hisoblanadi.

**Qurilish sohasida raqamlashtirish:**

O'zbekiston qurilish sohasi jadallik bilan rivojlanmoqda. Har yili minglab yangi turar-joy majmualari, savdo markazlari, zavod va fabrikalar qurilmoqda. Biroq, qurilish kompaniyalarining ko'pchiligi hali ham an'anaviy usulda ishlaydi va raqamli texnologiyalardan to'liq foydalanmayapti.

**Statistika:**
- O'zbekistonda 5000 dan ortiq ro'yxatga olingan qurilish kompaniyalari mavjud
- Ularning atigi 30% professional web saytga ega
- Faqat 15% onlayn xizmatlar (so'rov, hisob-kitob) ni taklif etadi
- 10% ijtimoiy tarmoqlarda faol

Bu ko'rsatkichlar qurilish sohasida raqamlashtirish uchun katta potentsial mavjudligini ko'rsatadi.

#### 1.1.3. Qurilish kompaniyalari uchun web saytning zaruriyati

Zamonaviy qurilish kompaniyasi uchun web sayt - bu shunchaki "bo'lsa yaxshi" narsa emas, balki biznesning muhim qismi. Keling, buni batafsil ko'rib chiqaylik:

**1. Birinchi taassurot**

Mijoz kompaniya haqida birinchi ma'lumotni ko'pincha internetdan oladi. Agar web sayt yo'q yoki yomon ishlangan bo'lsa, mijoz kompaniya professionalligiga shubha qiladi. Aksincha, zamonaviy va funksional sayt ishonch uyg'otadi.

**2. Loyihalarni namoyish etish**

Qurilish sohasida natijalar jismoniy ko'rinishga ega - binolar, inshootlar. Web sayt orqali:
- Bajarilgan loyihalar fotosuratlarini namoyish etish
- Loyiha jarayonini bosqichma-bosqich ko'rsatish
- Video-turlar taqdim etish
- 3D-vizualizatsiyalarni joylashtirish

**3. Xizmatlar to'g'risida to'liq ma'lumot**

Web saytda kompaniyaning barcha xizmatlari batafsil yoritilishi mumkin:
- Turli xil qurilish ishlari (turar-joy, sanoat, infratuzilma)
- Ta'mirlash va rekonstruksiya xizmatlari
- Loyihalash va arxitektura xizmatlari
- Maslahat xizmatlari

**4. Onlayn aloqa va so'rovlar**

Mijozlar uchun qulay:
- So'rov shakllari
- Onlayn chat
- Email orqali murojaat
- Qo'ng'iroq buyurtma qilish
- WhatsApp/Telegram integratsiyasi



**5. Marketing va SEO**

Web sayt - eng samarali marketing vositasi:
- Google, Yandex kabi qidiruv tizimlarida chiqish
- Kontekstli reklama (Google Ads, Yandex.Direct)
- Ijtimoiy tarmoqlar bilan integratsiya
- Email marketing uchun baza shakllantirish
- Kontent marketing (maqolalar, maslahatlar)

**6. Raqobat afzalligi**

Agar sizning kompaniyangiz web saytga ega, lekin raqibingizda yo'q bo'lsa - bu katta ustunlik. Mijoz internet qidiruvida sizni topadi, raqibingizni esa yo'qotib qo'yadi.

### 1.2. Loyihaning maqsadi va vazifalari

Web sayt quyidagi talablarga javob berishi kerak:
- Zamonaviy dizayn va professional ko'rinish
- Barcha qurilmalarda (kompyuter, planshet, telefon) to'g'ri ishlash
- Tez yuklanish va yuqori ishlash tezligi
- Xavfsizlik va ishonchlilik

   - Qulay navigatsiya tuzilmasini ishlab chiqish

**B. Loyihalash vazifalari:**

**C. Ishlab chiqish vazifalari:**

7. **Frontend dasturlash**

   - Server qismini yaratish

   - Autentifikatsiya va avtorizatsiya tizimini joriy etish

   - Backup strategiyasini joriy etish

10. **CMS integratsiyasi**
    - Administrator panelini yaratish
    - CRUD (Create, Read, Update, Delete) operatsiyalarini amalga oshirish
    - Foydalanuvchilar rollarini boshqarish
    - Kontent moderatsiya tizimini yaratish



**D. Sinovlar va optimallashtirish vazifalari:**

11. **Funksional testlar**

12. **Ishlash tezligini optimallashtirish**

13. **SEO optimallashtirish**
    - Meta teglar qo'shish

    - Sitemap yaratish

CREATE TABLE projects (

    title VARCHAR(200) NOT NULL,

    budget DECIMAL(15, 2),

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

);

CREATE TABLE project_images (
    id INT AUTO_INCREMENT PRIMARY KEY,
    project_id INT NOT NULL,
    image_url VARCHAR(255) NOT NULL,
    is_main BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    icon VARCHAR(255),
    price_from DECIMAL(10, 2)
);

-- Index'lar yaratish (tezlik uchun)
CREATE INDEX idx_category ON projects(category);
CREATE INDEX idx_status ON projects(status);

SELECT 
    p.id,
    p.title,
    p.description,
    p.category,
    p.budget,
    COUNT(pi.id) as image_count,
    GROUP_CONCAT(pi.image_url) as images
FROM projects p
LEFT JOIN project_images pi ON p.id = pi.project_id
WHERE p.status = 'completed'
GROUP BY p.id
ORDER BY p.created_at DESC

// src/controllers/projectController.ts
import { Request, Response, NextFunction } from 'express';
import { PrismaClient } from '@prisma/client';
import { uploadImages } from '../services/uploadService';

const prisma = new PrismaClient();

export const getProjects = async (
  req: Request,
  res: Response,
  next: NextFunction
) => {
  try {
    const { category, status, page = 1, limit = 12 } = req.query;

    const where: any = {};
    if (category) where.category = { slug: category };
    if (status) where.status = status;

    const skip = (Number(page) - 1) * Number(limit);

    const [projects, total] = await Promise.all([
      prisma.project.findMany({
        where,
        include: {
          category: true,
          images: {
            orderBy: { order: 'asc' },
          },
        },
        skip,
        take: Number(limit),
        orderBy: { createdAt: 'desc' },
      }),
      prisma.project.count({ where }),
    ]);

    res.json({
      success: true,
      data: projects,
      pagination: {
        page: Number(page),
        limit: Number(limit),
        total,
        pages: Math.ceil(total / Number(limit)),
      },
    });
  } catch (error) {
    next(error);
  }
};

export const getProjectById = async (
  req: Request,
  res: Response,
  next: NextFunction
) => {
  try {
    const { id } = req.params;

    const project = await prisma.project.findUnique({
      where: { id: Number(id) },
      include: {
        category: true,
        images: {
          orderBy: { order: 'asc' },
        },
      },
    });

    if (!project) {
      return res.status(404).json({
        success: false,
        message: 'Loyiha topilmadi',
      });
    }

    // Ko'rishlar sonini oshirish
    await prisma.project.update({
      where: { id: Number(id) },
      data: { views: { increment: 1 } },
    });

    res.json({
      success: true,
      data: project,
    });
  } catch (error) {
    next(error);
  }
};

export const createProject = async (
  req: Request,
  res: Response,
  next: NextFunction
) => {
  try {
    const { title, description, categoryId, budget, area, duration, location } = req.body;
    const files = req.files as Express.Multer.File[];

    // Slug yaratish
    const slug = title
      .toLowerCase()
      .replace(/[^a-z0-9]+/g, '-')
      .replace(/(^-|-$)/g, '');

    // Rasmlarni yuklash
    const imageUrls = await uploadImages(files);

    // Loyiha yaratish
    const project = await prisma.project.create({
      data: {
        title,
        slug,
        description,
        categoryId: Number(categoryId),
        budget: budget ? Number(budget) : null,
        area: area ? Number(area) : null,

// src/middleware/auth.ts
import { Request, Response, NextFunction } from 'express';
import jwt from 'jsonwebtoken';

const JWT_SECRET = process.env.JWT_SECRET || 'your-secret-key';

interface JwtPayload {
  userId: number;
  email: string;
  role: string;
}

declare global {
  namespace Express {
    interface Request {
      user?: JwtPayload;
    }
  }
}

export const authenticate = (
  req: Request,
  res: Response,
  next: NextFunction
) => {
  try {
    const token = req.headers.authorization?.replace('Bearer ', '');

    if (!token) {
      return res.status(401).json({
        success: false,
        message: 'Token topilmadi',
      });
    }

    const decoded = jwt.verify(token, JWT_SECRET) as JwtPayload;
    req.user = decoded;
    next();
  } catch (error) {
    return res.status(401).json({
      success: false,
      message: 'Noto\'g\'ri yoki muddati o\'tgan token',
    });
  }
};

export const authorize = (roles: string[]) => {
  return (req: Request, res: Response, next: NextFunction) => {
    if (!req.user) {
      return res.status(401).json({
        success: false,
        message: 'Autentifikatsiya talab qilinadi',
      });
    }

    if (!roles.includes(req.user.role)) {
      return res.status(403).json({
        success: false,
        message: 'Ruxsat yo\'q',
      });
    }

    next();
  };
};

import sharp from 'sharp';

import fs from 'fs';

export const uploadImages = async (files: Express.Multer.File[]): Promise<string[]> => {
  const processedImages: string[] = [];

  for (const file of files) {
    const filename = file.filename;
    const filepath = file.path;
    
    // Thumbnail yaratish
    const thumbnailPath = path.join(
      path.dirname(filepath),
      'thumbnails',
      filename
    );
    
    // Thumbnail papkasini yaratish
    const thumbnailDir = path.dirname(thumbnailPath);
    if (!fs.existsSync(thumbnailDir)) {
      fs.mkdirSync(thumbnailDir, { recursive: true });
    }

    // Rasmni optimallashtirish
    await sharp(filepath)
      .resize(1920, 1080, {
        fit: 'inside',
        withoutEnlargement: true,
      })
      .jpeg({ quality: 85 })
      .toFile(filepath + '.optimized');

    // Original faylni o'chirish va optimallashtirilgan bilan almashtirish
    fs.unlinkSync(filepath);
    fs.renameSync(filepath + '.optimized', filepath);

    // Thumbnail yaratish
    await sharp(filepath)
      .resize(400, 300, { fit: 'cover' })
      .jpeg({ quality: 80 })
      .toFile(thumbnailPath);

    processedImages.push(`/uploads/${filename}`);
  }

  return processedImages;
};

export const cacheMiddleware = (duration: number) => {
  return (req: any, res: any, next: any) => {
